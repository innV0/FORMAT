# About _FORMAT

_FORMAT stands for **Flat, Omnipotent, Readable, Model, Annotated, Template**.

It is an open-source specification for structured Markdown where every file carries its own schema. Define concepts, relationships, and evaluations in plain text — whether you're modeling a business strategy, documenting a procedure, or building a domain-specific knowledge model.

## Why

Traditional modeling tools store data in opaque formats. This creates problems:

- **No portability** — models can't be shared without dedicated software
- **Poor diffing** — database exports produce noisy diffs
- **AI friction** — LLMs struggle with complex relational structures

_FORMAT solves these by keeping everything in a single, self-describing Markdown file. The schema travels with the data.

## Templates

_FORMAT ships with two official templates:

- **Business** — stakeholders, segments, value propositions, finances, team, strategy (70+ concepts, 13 matrices)
- **Procedures** — workflows, ordered steps, roles, artifacts, tools, branching logic

Each template is a complete, self-contained schema. You can also define your own.
